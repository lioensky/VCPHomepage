# VCP 浪潮 v8.2 · 语义流体力学可视化

> 一个独立的纯 HTML / Canvas / anime.js 单页项目，用"河流"的比喻讲清楚 VCP TagMemo 浪潮 v8.2 引擎到底是怎么工作的。

## 它在讲什么故事

把 RAG 想象成水利工程：

- 每篇日记里的 Tag 序列 = 一条**从前往后流的河**
- 同一个 Tag 在多篇日记中出现 = 形成**支流网络**（共现拓扑）
- 顺流容易、逆流要付能量 = **V8.2 有序双向势能**
- 同义词回音 / 偶然共现 = 用**钟形阻尼**抑制
- 传统 RAG 算两点直线 cosine = 穿山而过
- VCP 算流体力学最优河道 = **测地线重排**（黎曼曲率）
- 河道阻力 = **LIF 神经元电流**算法
- 河道落差太大产生浪花 = **虫洞路由**
- AI 修建的运河跨域 = **朗飞结跳跃**
- 残差金字塔 = **地势等高线图**（O(n³) 预计算）
- SVD = 河流区域分析图（O(n³) 离线）
- 流体寻址 = **O(1) 查表**（运行时零开销）

## 怎么打开

直接双击 `index.html`，或起一个本地静态服务器（推荐，避免某些浏览器对 file:// 的 Canvas 限制）：

```bash
# Python 3
python -m http.server 8080

# Node (需要 npx)
npx serve .

# 或 VS Code 的 Live Server 插件
```

然后访问 `http://localhost:8080/`。

## 交互点

- **右侧导航圆点**：跳转到任意章节
- **Ch3**：切换 顺流 / 逆流 模式
- **Ch6**：点击「注入脉冲」手动触发 LIF 扩散
- **Ch7**：点击「触发虫洞」手动触发飞溅
- 多数场景有自动循环触发，每隔几秒自播放一次

## 技术细节

- **anime.js** 3.2.1（CDN）：负责文本入场、统计数字递增、按钮点击反馈
- **Canvas 2D**：所有河流 / 粒子 / 等高线全部用 Canvas 绘制
- **HiDPI 适配**：自动检测 `devicePixelRatio`，避免 Retina 屏模糊
- **IntersectionObserver**：进入视口才启动 RAF，离开自动停止，省 CPU
- **零打包构建**：纯静态文件，可直接部署到任意静态托管

## 设计哲学

每一帧都试图回答："如果你不写代码，能不能在 30 秒内 get 到这一章的核心？"

> 比喻不是降级，是另一种精确。
> "顺流 1.0、逆流 0.42" 的数字背后，是河水真的需要更多能量才能逆向流动。
